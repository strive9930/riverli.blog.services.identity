# 登录接口升级集成指南

## 📋 概述

身份认证服务已升级，登录接口现在一次性返回完整的用户信息、角色、权限和菜单数据。本文档说明如何在前端项目 `river-blog-frontend-admin` 中集成这些改进。

## 🎯 改进内容

### 之前的流程
```
1. POST /api/auth/login → 获取 token
2. GET /api/user-frontend/info → 获取用户信息
3. GET /api/user-frontend/permissions → 获取权限列表
4. GET /api/user-frontend/menus → 获取菜单树
5. GET /api/user-frontend/roles → 获取角色列表
```

### 现在的流程
```
1. POST /api/auth/login → 获取所有数据（token + 用户信息 + 角色 + 权限 + 菜单）
```

**优势：**
- ✅ 减少 4 次网络请求
- ✅ 提升登录速度
- ✅ 简化前端逻辑
- ✅ 更好的用户体验

### 2. 更新 User Store

修改 `src/store/modules/user.ts`：

```typescript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import service from '@/utils/request'
import { ElMessage } from 'element-plus'
import type { LoginResponseData, UserInfo, RoleInfo, MenuTreeItem } from '@/types/auth'

export const useUserStore = defineStore('user', () => {
  // 状态
  const accessToken = ref<string>(localStorage.getItem('access_token') || '')
  const userInfo = ref<UserInfo | null>(
    (() => {
      const stored = localStorage.getItem('user_info')
      if (!stored || stored === 'undefined') return null
      try {
        return JSON.parse(stored)
      } catch (e) {
        console.warn('用户信息JSON解析失败:', e)
        return null
      }
    })()
  )

  // 计算属性
  const isLoggedIn = computed(() => !!accessToken.value)
  const email = computed(() => userInfo.value?.email || '')
  const nickName = computed(() => userInfo.value?.nickName || userInfo.value?.nickname || '')
  const avatar = computed(() => userInfo.value?.avatar || '')
  const roles = computed(() => userInfo.value?.roles || [])
  const permissions = computed(() => userInfo.value?.permissions || [])
  const menus = computed(() => userInfo.value?.menus || [])
  const isAdmin = computed(() => userInfo.value?.isAdmin || false)
  const roleNames = computed(() => roles.value.map(r => r.name))

  // Actions
  const login = async (credentials: { email: string; password: string }): Promise<{ success: boolean; message?: string }> => {
    console.log('开始登录流程，凭据:', credentials);
    try {
      const response = await service.post('/auth/login', credentials);
      console.log('收到登录响应:', response);
      
      // 检查根级别的success字段
      if (response.success && response.data) {
        const loginData: LoginResponseData = response.data;
        
        if (!loginData.token) {
          console.error('登录数据格式错误:', loginData);
          return { success: false, message: '登录数据格式错误' };
        }
        
        // 构建用户信息对象
        const newUserInfo: UserInfo = {
          id: loginData.userId,
          username: loginData.username,
          email: loginData.email,
          nickName: loginData.nickname,
          avatar: loginData.avatar,
          roles: loginData.roles,
          permissions: loginData.permissions,
          menus: loginData.menus,
          isAdmin: loginData.isAdmin,
          createdAt: loginData.createdAt
        };
        
        console.log('设置的用户信息:', newUserInfo);
        
        // 存储token和用户信息
        accessToken.value = loginData.token;
        userInfo.value = newUserInfo;
        
        // 持久化到localStorage
        localStorage.setItem('access_token', accessToken.value);
        localStorage.setItem('user_info', JSON.stringify(userInfo.value));
        
        console.log('登录成功，用户信息已保存');
        
        return { success: true };
      } else {
        console.log('登录失败:', response.message);
        return { success: false, message: response.message || '登录失败' };
      }
    } catch (error: any) {
      console.error('登录失败:', error)
      return { 
        success: false, 
        message: error.response?.data?.message || '登录失败，请检查网络连接' 
      }
    }
  }

  const logout = () => {
    // 清除所有用户相关数据
    accessToken.value = ''
    userInfo.value = null
    
    // 清除localStorage
    localStorage.removeItem('access_token')
    localStorage.removeItem('user_info')
    localStorage.removeItem('routes')
    localStorage.removeItem('menus')
    localStorage.removeItem('menuTree')
    
    ElMessage.success('退出登录成功')
  }

  const initUserState = async () => {
    console.log('=== 开始初始化用户状态 ===')
    const storedToken = localStorage.getItem('access_token')
    const storedUserInfo = localStorage.getItem('user_info')
    
    if (storedToken && storedUserInfo && storedUserInfo !== 'undefined') {
      accessToken.value = storedToken
      try {
        userInfo.value = JSON.parse(storedUserInfo)
        console.log('从localStorage恢复用户状态成功')
      } catch (e) {
        console.warn('初始化用户状态时JSON解析失败:', e)
        logout()
      }
    } else {
      console.log('没有找到存储的用户状态')
    }
    console.log('=== 用户状态初始化完成 ===')
  }

  return {
    // 状态
    accessToken,
    userInfo,
    
    // 计算属性
    isLoggedIn,
    email,
    nickName,
    avatar,
    roles,
    roleNames,
    permissions,
    menus,
    isAdmin,
    
    // 方法
    login,
    logout,
    initUserState
  }
})
```

### 3. 更新路由守卫

修改 `src/router/index.ts` 或相关的路由守卫文件：

```typescript
import router from './index'
import { useUserStore } from '@/store/modules/user'
import { usePermissionStore } from '@/store/modules/permission'

router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore()
  const permissionStore = usePermissionStore()
  
  if (userStore.isLoggedIn) {
    if (to.path === '/login') {
      next({ path: '/' })
    } else {
      // 如果还没有加载动态路由，则加载
      if (!permissionStore.isRoutesLoaded) {
        try {
          // 使用登录时获取的菜单数据生成路由
          await permissionStore.generateRoutes(userStore.menus)
          next({ ...to, replace: true })
        } catch (error) {
          console.error('生成路由失败:', error)
          userStore.logout()
          next(`/login?redirect=${to.path}`)
        }
      } else {
        next()
      }
    }
  } else {
    if (to.path === '/login') {
      next()
    } else {
      next(`/login?redirect=${to.path}`)
    }
  }
})
```

### 4. 更新 Permission Store

修改 `src/store/modules/permission.ts`：

```typescript
import { defineStore } from 'pinia'
import { ref } from 'vue'
import type { MenuTreeItem } from '@/types/auth'
import type { RouteRecordRaw } from 'vue-router'

export const usePermissionStore = defineStore('permission', () => {
  const routes = ref<RouteRecordRaw[]>([])
  const isRoutesLoaded = ref(false)

  // 将菜单树转换为路由配置
  const generateRoutes = async (menus: MenuTreeItem[]) => {
    const dynamicRoutes: RouteRecordRaw[] = []
    
    menus.forEach(menu => {
      const route = convertMenuToRoute(menu)
      if (route) {
        dynamicRoutes.push(route)
      }
    })
    
    routes.value = dynamicRoutes
    isRoutesLoaded.value = true
    
    // 添加到路由器
    dynamicRoutes.forEach(route => {
      router.addRoute(route)
    })
    
    return dynamicRoutes
  }

  // 转换菜单项为路由配置
  const convertMenuToRoute = (menu: MenuTreeItem): RouteRecordRaw | null => {
    if (!menu.path) return null
    
    const route: RouteRecordRaw = {
      path: menu.path,
      name: menu.name,
      meta: {
        title: menu.title,
        icon: menu.icon,
        permissions: menu.requiredPermission ? [menu.requiredPermission] : []
      }
    }
    
    // 如果有子菜单，递归处理
    if (menu.children && menu.children.length > 0) {
      route.children = menu.children
        .map(child => convertMenuToRoute(child))
        .filter((r): r is RouteRecordRaw => r !== null)
    }
    
    return route
  }

  const resetRoutes = () => {
    routes.value = []
    isRoutesLoaded.value = false
  }

  return {
    routes,
    isRoutesLoaded,
    generateRoutes,
    resetRoutes
  }
})
```

### 5. 更新登录页面

修改 `src/views/login/index.vue`：

```vue
<template>
  <div class="login-container">
    <el-form :model="loginForm" :rules="rules" ref="loginFormRef">
      <el-form-item prop="email">
        <el-input v-model="loginForm.email" placeholder="邮箱" />
      </el-form-item>
      <el-form-item prop="password">
        <el-input v-model="loginForm.password" type="password" placeholder="密码" />
      </el-form-item>
      <el-button type="primary" @click="handleLogin" :loading="loading">
        登录
      </el-button>
    </el-form>
  </div>
</template>

<script setup lang="ts">
import { ref, reactive } from 'vue'
import { useRouter, useRoute } from 'vue-router'
import { useUserStore } from '@/store/modules/user'
import { ElMessage } from 'element-plus'

const router = useRouter()
const route = useRoute()
const userStore = useUserStore()

const loginFormRef = ref()
const loading = ref(false)

const loginForm = reactive({
  email: '',
  password: ''
})

const rules = {
  email: [{ required: true, message: '请输入邮箱', trigger: 'blur' }],
  password: [{ required: true, message: '请输入密码', trigger: 'blur' }]
}

const handleLogin = async () => {
  if (!loginFormRef.value) return
  
  await loginFormRef.value.validate(async (valid: boolean) => {
    if (valid) {
      loading.value = true
      try {
        const result = await userStore.login(loginForm)
        
        if (result.success) {
          ElMessage.success('登录成功')
          
          // 登录后直接跳转，不需要再获取用户信息
          const redirect = (route.query.redirect as string) || '/'
          router.push(redirect)
        } else {
          ElMessage.error(result.message || '登录失败')
        }
      } catch (error) {
        ElMessage.error('登录失败，请稍后重试')
      } finally {
        loading.value = false
      }
    }
  })
}
</script>
```

## 🐛 常见问题

### Q1: 登录后菜单不显示？
**A:** 检查以下几点：
1. 确认后端返回的 `menus` 字段不为空
2. 检查 `permissionStore.generateRoutes()` 是否正确调用
3. 查看浏览器控制台是否有路由注册错误

### Q2: 权限判断不生效？
**A:** 确保：
1. `userInfo.permissions` 数组正确填充
2. 权限指令或函数使用的是正确的权限码
3. 权限码与后端定义一致

### Q3: 页面刷新后丢失用户状态？
**A:** 检查：
1. `localStorage` 中是否正确保存了 `user_info`
2. `initUserState()` 是否在应用启动时调用
3. JSON 解析是否成功
