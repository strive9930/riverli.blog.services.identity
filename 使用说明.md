# River Blog 身份认证与权限管理系统使用说明

## 📋 系统概述

本系统基于 ASP.NET Core 和 Vue 3 构建，采用 DDD 分层架构实现完整的 RBAC（基于角色的访问控制）权限管理体系。系统包含用户管理、角色管理、权限管理和菜单管理四大核心模块。

## 🏗️ 技术架构

### 后端技术栈
- **框架**: ASP.NET Core 8.0
- **架构**: DDD（领域驱动设计）分层架构
- **ORM**: Entity Framework Core
- **数据库**: MySQL
- **认证**: JWT + ASP.NET Core Identity
- **权限模型**: RBAC（角色基础访问控制）
- **消息处理**: MediatR CQRS 模式

### 前端技术栈
- **框架**: Vue 3 + TypeScript
- **UI库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **构建工具**: Vite

## 🚀 系统部署

### 后端部署

1. **环境准备**
   ```bash
   # 确保安装 .NET 8.0 SDK
   dotnet --version
   
   # 确保MySQL服务运行
   mysql --version
   ```

2. **数据库配置**
   ```bash
   # 修改 appsettings.json 中的连接字符串
   "ConnectionStrings": {
     "DefaultConnection": "Server=localhost;Port=3306;Database=RiverBlog_Identity;Uid=root;Pwd=your_password;"
   }
   ```

3. **运行应用**
   ```bash
   cd RiverLi.Blog.Services.Identity.Api
   dotnet run
   # 或使用开发模式
   dotnet watch run
   ```

4. **初始数据**
   - 系统启动时会自动创建管理员账号：`admin@example.com` / `Admin123!`
   - 自动初始化权限、角色和菜单数据

### 前端部署

1. **环境准备**
   ```bash
   # 确保安装 Node.js 16+
   node --version
   npm --version
   ```

2. **安装依赖**
   ```bash
   cd river-blog-frontend-admin
   npm install
   ```

3. **配置后端API地址**
   ```bash
   # 修改 .env.development 文件
   VITE_API_BASE_URL=http://localhost:5001
   ```

4. **运行开发服务器**
   ```bash
   npm run dev
   # 访问地址: http://localhost:3001
   ```

5. **构建生产版本**
   ```bash
   npm run build
   ```

## 👥 用户管理

### 登录系统

1. **管理员登录**
   - 用户名: `admin@example.com`
   - 密码: `Admin123!`

2. **普通用户登录**
   - 可通过注册功能创建新用户
   - 或由管理员在用户管理界面创建

### 用户操作

#### 创建用户
1. 进入「用户管理」→「用户列表」
2. 点击「新增用户」按钮
3. 填写用户基本信息：
   - 用户名（必填）
   - 邮箱（必填，用于登录）
   - 密码（必填）
   - 昵称
   - 手机号
4. 选择用户角色
5. 点击「确定」完成创建

#### 编辑用户
1. 在用户列表中找到目标用户
2. 点击「编辑」按钮
3. 修改相关信息
4. 点击「确定」保存更改

#### 删除用户
1. 在用户列表中勾选要删除的用户
2. 点击「批量删除」按钮
3. 确认删除操作

#### 重置密码
1. 在用户列表中点击目标用户的「更多」按钮
2. 选择「重置密码」
3. 设置新密码并确认

## 🎭 角色管理

### 角色概念
角色是权限的集合体，用户通过角色获得相应的系统访问权限。

### 创建角色

1. 进入「角色管理」→「角色列表」
2. 点击「新增角色」按钮
3. 填写角色信息：
   ```
   角色名称: 系统管理员
   角色编码: SYSTEM_ADMIN
   描述: 拥有系统最高权限
   状态: 启用
   ```
4. 点击「下一步」进行权限分配

### 权限分配

#### 方式一：分组分配（推荐）
1. 在权限分配界面，按权限分组查看
2. 展开相应分组（如「用户管理」、「菜单管理」等）
3. 勾选需要的权限项
4. 点击「保存」完成分配

#### 方式二：搜索分配
1. 使用搜索框查找特定权限
2. 直接勾选搜索结果中的权限
3. 批量保存权限分配

#### 方式三：继承分配
1. 选择父级角色
2. 子角色将继承父角色的部分或全部权限
3. 可在此基础上增减权限

### 角色操作

#### 编辑角色
1. 在角色列表中点击「编辑」
2. 可修改角色基本信息和权限分配
3. 保存更改后立即生效

#### 删除角色
1. 确保该角色下没有用户
2. 在角色列表中点击「删除」
3. 确认删除操作

#### 查看角色详情
1. 点击角色列表中的「查看」按钮
2. 可查看角色的基本信息、权限列表和关联用户

## 🔐 权限管理

### 权限结构

权限按功能模块分组：
- **仪表盘权限**: dashboard.view
- **用户管理权限**: users.view, users.create, users.edit, users.delete
- **角色管理权限**: roles.view, roles.create, roles.edit, roles.delete
- **权限管理权限**: permissions.view, permissions.create, permissions.edit, permissions.delete
- **菜单管理权限**: menus.view, menus.create, menus.edit, menus.delete
- **内容管理权限**: articles.view, articles.create, articles.edit, articles.delete
- **数据分析权限**: analytics.overview.view, analytics.users.view

### 权限操作

#### 创建自定义权限
1. 进入「权限管理」界面
2. 点击「新增权限」
3. 填写权限信息：
   ```
   权限名称: 自定义报表查看
   权限编码: reports.custom.view
   所属分组: 数据分析
   描述: 允许查看自定义报表
   ```
4. 保存权限

#### 权限分组管理
1. 权限按功能模块自动分组
2. 可通过分组快速查找和分配权限
3. 支持权限的批量操作

#### 权限状态管理
- **启用**: 权限正常使用
- **禁用**: 权限暂时不可用，不影响已分配的角色

## 📋 菜单管理

### 菜单类型

1. **普通菜单项**: 对应具体的功能页面
2. **分组标题**: 用于菜单分类，不对应具体页面
3. **分隔符**: 在菜单中起分隔作用
4. **外部链接**: 跳转到外部网站

### 创建菜单项

#### 基本菜单项
1. 进入「菜单管理」→「菜单管理」
2. 点击「新增菜单」
3. 填写菜单信息：
   ```
   菜单名称: ReportManagement
   显示标题: 报表管理
   路由路径: /reports
   图标: Document
   排序: 5
   菜单类型: 普通菜单项
   所属菜单组: 数据分析
   所需权限: reports.view
   描述: 系统报表管理功能
   ```
4. 设置状态（启用/可见）
5. 点击「确定」保存

#### 父子菜单结构
1. 创建父级菜单（通常为分组标题类型）
2. 创建子菜单时，在「父级菜单」选项中选择对应的父菜单
3. 系统会自动构建层级结构

### 菜单组管理

#### 创建菜单组
1. 进入「菜单管理」→「菜单组管理」
2. 点击「新增菜单组」
3. 填写信息：
   ```
   菜单组名称: 系统管理
   编码: SYSTEM
   图标: Setting
   排序: 1
   描述: 系统基础管理功能
   ```
4. 保存菜单组

#### 管理菜单组
- 可对菜单组进行编辑、启用/禁用、删除操作
- 删除菜单组前需确保组内无菜单项

### 前端路由绑定

菜单项通过以下方式与前端路由关联：

1. **路由路径匹配**: 菜单的 `path` 字段对应 Vue Router 的路径
2. **权限控制**: `requiredPermission` 字段控制访问权限
3. **动态路由生成**: 系统根据用户权限动态生成可访问路由

示例配置：
```json
{
  "name": "UserManagement",
  "title": "用户管理",
  "path": "/users",
  "icon": "User",
  "requiredPermission": "users.view",
  "menuType": 1,
  "menuGroupId": "system-group-id"
}
```

## 🔧 开发指南

## 🛠️ 常见问题

### 登录问题
**问题**: 登录时报401错误
**解决方案**: 
1. 检查后端服务是否正常运行
2. 验证数据库连接是否正常
3. 确认用户账号密码正确

### 权限问题
**问题**: 有权限但页面无法访问
**解决方案**:
1. 检查菜单管理中是否正确配置了路由路径
2. 验证权限编码是否与后端一致
3. 刷新页面重新加载权限

### 菜单不显示
**问题**: 左侧菜单栏空白
**解决方案**:
1. 检查网络请求是否正常
2. 查看浏览器控制台是否有错误
3. 确认用户是否拥有相应权限

### 数据库连接失败
**问题**: 启动时报数据库连接错误
**解决方案**:
1. 检查MySQL服务是否运行
2. 验证连接字符串配置
3. 确认数据库用户权限
